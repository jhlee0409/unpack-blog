# 1. DApp

- `Decentralized Application` (탈중앙화 애플리케이션)의 약자이다.
- `블록체인` or `P2P`(peer to peer) 네트워크를 기반으로한 애플리케이션이다.
- `P2P` : 컴퓨터 간의 네트워크
- 한 기관의 영역과 제어, 통제에서 벗어난 것, 유저들끼리 스마트 계약을 통해 연결

# 2. 탈중앙화 vs 중앙화

- 그렇다면 `탈중앙화`와 `중앙화`의 차이점은 무엇일까?

## 2-1. 중앙화

- 대부분의 오래된 금융기관에서는 중앙화 시스템을 이용하고 있다.
- 이는 특정 기관을 통해 사용자 데이터를 저장, 관리하는 것을 말한다.
- 모든 사용자가 하나의 데이터 소스에 연결이 되어있다.
- 이런 관계를 중앙화 네트워크라 한다.

### 단점

- `단일 장애점`([SPOF](https://ko.wikipedia.org/wiki/%EB%8B%A8%EC%9D%BC_%EC%9E%A5%EC%95%A0%EC%A0%90))가 생긴다.
- 중앙화 DB에 문제가 생기면 모든 데이터에 악영향을 끼치며, 전부 사라질 수 도 있다.
- 이유는 하나의 기관이 통제, 관리, 저장을 하기 때문이다.

#

## 2-2. 탈중앙화

- 네트워크의 각기 다른 노드와 컴퓨터에 데이터가 저장됨
- 어떤 노드도 중앙 기관이 관리하지 않는다.
- 모든 노드가 서로 신뢰할 수 있어야하니 상호 간 동의할 수 있는 네트워크가 필요하다.
- 모든 노드는 만들어질때 똑같은 DB의 `복사본`을 가지게 된다.
- 하나의 노드에 문제가 생겨도 다른 노드가 정보를 들고 있기 때문에 데이터 복구가 가능하며. `수정이나 삭제가 불가능` 하다. 오로직 `추가`만 된다.
- 그렇기에 데이터를 변형해도 다른 노드들이 원본의 복제본을 들고 있기 떄문에 불가능하다.
- 그렇기에 중앙화 시스템보다 더 `보안상` 뛰어나고 `신뢰도`가 높다.
